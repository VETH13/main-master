# 游戏设计文档（简短版） — Ritual 链集成 2D 平台原型

版本：v0.1
目标：基于 Godot 的 2D 平台小游戏，Web（HTML5）发布。支持在 Ritual 链测试网上“连接钱包并显示地址/余额”，未来可扩展为链上铸造/记录成绩。

核心理念
- 轻量、有趣、易上手：单关卡平台通关为目标，玩家控制角色到达终点取得高分。
- 链集成为可选模块：默认只是钱包连接与只读展示（地址、代币余额）。写入/铸造行为保留示例代码，由用户确认后在测试网执行以避免意外花费。

MVP 功能（1 周目标）
- 游戏内玩法
  - 角色移动：左右移动、跳跃、重生（碰到陷阱回到起点）
  - 障碍与简单敌人（可用占位方块）
  - 一个可通关关卡（起点 → 终点）
  - 得分（收集物或通关时间）与简单 UI（分数/生命/重试）
- 链交互（页面层）
  - 页面右上角“Connect Wallet”按钮（ethers.js）
  - 连接后显示钱包地址与 Ritual 测试网余额（read-only，调用 provider.getBalance 和可选 ERC-20 balance）
  - 当玩家通关时，弹出提示：“记录成绩到区块链（需要签名）”——若玩家选择，可执行示例交易（需要额外确认与 gas）
- 文档
  - README（本地运行/导出步骤）
  - 如何把 Ritual 测试网参数加入 MetaMask 的示例（placeholder，需替换为 Ritual 官方参数）
  - 智能合约接口草案（ABI 示例 + JS 调用示例）——放在 repo 以便将来扩展

未来扩展（非 MVP）
- 通关铸造 NFT（证明/徽章）
- 链上排行榜（分数哈希上链 + off-chain 排行索引）
- 道具/皮肤以 ERC-1155 管理
- 支付解锁关卡（链上支付或 Layer2）

安全与成本考虑
- 默认不开写交易。任何上链写入需手动触发并展示估算 gas/费用���
- 使用测试网开发与演示，减少成本与风险。
- 推荐将大数据（录像、完整分数表）放 off-chain，仅写哈希或凭证上链。

技术选型
- 引擎：Godot 4（2D），导出 HTML5
- 链接层：ethers.js（网页端） + 浏览器钱包（MetaMask / WalletConnect）
- 部署：Web 静态托管（GitHub Pages / Netlify 等）或本地静态服务（测试）
